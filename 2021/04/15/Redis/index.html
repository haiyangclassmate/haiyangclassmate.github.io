Redis命令

Redis处理数据是单线程的

  • Redis压缩包位置 —– cd /home/soft/
  • Redis存放位置 —– cd /usr/local/redis-3.2.9
  • Redis启动服务、开始工作 —— cd /usr/local/redis-3.2.9/src
    • redis-server 启动服务
    • redis-cli 启动客户端
  • 前台启动Redis —– ./redis-server
  • 后台启动Redis —– ./redis-server &
    • 后台启动的前提是修改 redis.conf 配置文件，设置后台启动redis服务
    • 把文件中的==daemonize==属性改为==yes==（表明需要在后台运行）
  • 关闭Redis —– ./redis-cli shutdown
  • 关闭Redis —– kill -9 pid
  • 查看Redis进程 —– ps -ef | grep redis
  • ==cmd查看Redis端口是否开放 —— telnet 云ip 6379==
  • 查看Redis进程和端口 —– netstat -an |grep 6379
  • 开启Redis防火墙 —– systemctl start firewalld
  • 查看防火墙状态（正在运行还是停止）：—– systemctl status firewalld
  • 如果未开放6379端口则执行 —— firewall-cmd –zone=public –add-port=6379/tcp –permanent
  • 查看对应端口防火墙是否关闭： —– firewall-cmd –zone=public –query-port=端口号/tcp
  • ==查看已经开放的端口： —— firewall-cmd –list-all== （RDM远程连不上先看看6379的端口是否开放）

  • ping —— 返回pong表示Redis服务器正常运行

  • dbsize —– 返回第0个数据库的key数量

  • select XXX —— 选择其他数据库 （select 0：表示选择第0 个数据库）

  • flushdb —– 删除==当前库==的数据

  • exit 或者 quit —– 退出客户端（==没有退服务==）

  • 操作键的命令

  • keys * —– 查询所有的key

    • keys w* —– 查询key以w开头的字符（ wall、world、word、wood ）
    • keys wo?d —– 一个问号代表一个字符（ word、wood ）
    • keys wo??d —– 两个问号代表两个字符（ world ）

  • exists key —– 判断当前key是否存在，存在即返回数量

    • exists k1 —– 1
    • exists k1 k2 —– 2

  • expire key seconds —- 设置键的存活时间

    • expire k1 20 （表示设置k1的存活时长为20秒）
      • 设置成功返回数字1
      • 不成功返回数字0

  • ttl key —– 返回key的剩余时间

    • -1 ：没有设置 key 的生存时间， key 永不过期。
    • -2：key 不存在
    • 数字：key 的剩余时间，秒为单位

  • type key —— 返回值：字符串表示的数据类型

    • none (key 不存在)
    • string (字符串)
    • list (列表)
    • set (集合)
    • zset (有序集)
    • hash (哈希表)

  • del key [key1……] —– 删除存在的 key，不存在的 key 忽略。

    • 数字，删除的 key 的数量。

3. 事务

  • multi 标记事务的开始

  • exec 执行所有事务块内的命令

  • discard 主动放弃事务，放弃执行事务块内的所有命令

  • watch 监视一个(或多个) key ，如果在事务执行之前这个(或这些) key 被其他命令所改动，

    ​ 那么事务将被打断。任意时刻只有一个事务可以修改数据，保持数据一致。

注意：

  1. 事务执行exec之前，入队命令错误，语法错误，导致服务器不能正常工作的，会放弃事务，都不执行。
  2. 事务执行exec之前，语法没错，执行时出错，事务中已执行的命令是正常执行的。（redis中没有事务回滚）

4. 持久化

  • 把内存中的数据保存到磁盘中

持久化可以理解为存储，就是将数据存储到一个不会丢失的地方，如果把数据放在内存

中，电脑关闭或重启数据就会丢失，所以放在内存中的数据不是持久化的，而放在磁盘就算

是一种持久化。

Redis 的数据存储在内存中，内存是瞬时的，如果 linux 宕机或重启，又或者 Redis 崩溃

或重启，所有的内存数据都会丢失，为解决这个问题，Redis 提供两种机制对数据进行持久

化存储，便于发生故障后能迅速恢复数据。

4.1 RDB（快照方式）

​ Redis Database（RDB），就是在指定的时间间隔内将内存中的数据集快照写入磁盘，数

据恢复时将快照文件直接再读到内存。

RDB 保存了在某个时间点的数据集（全部数据）。存储在一个二进制文件中，只有一个

文件。默认是 dump.rdb。RDB 技术非常适合做备份，可以保存最近一个小时，一天，一个月

的全部数据。保存数据是在单独的进程中写文件，不影响 Redis 的正常使用。RDB 恢复数据

时比其他 AOF 速度快

==缺点：数据有丢失的可能性==

4.2 AOF

  • ==只记录对库的修改命令==

src下没有 appendonly.aof 文件时，修改上一级的redis.conf文件，将appendonly：默认是 no，改成 yes 即开启了 aof 持久化

5. 主从复制

5.1 主从复制-读写分离（master/slave）

实现步骤：

  1. 复制redis.conf 分别为redis6380.conf redis6380.conf redis6380.conf

  2. 分别启动三个配置文件服务 ps -ef | grep redis 查一下

  3. 指定端口连接服务器 ./redis-cli -p 6380

  4. 查看服务器信息 info replication

  5. 新开两个窗口 登录 6382端口和6384 端口查看服务器信息

==模拟容灾处理==

  • 当 Master 服务出现故障，需手动将 slave 中的一个提升为 master，剩下的 slave 挂至新的master 上（冷处理：机器挂掉了，再处理）

命令：

①：slaveof no one 将一台 slave 服务器提升为 Master （提升某 slave 为 master）

②：slaveof 127.0.0.1 6382（将 slave 挂至新的 master 上）

步骤：

  1. 将 Master:6380 停止（模拟挂掉）

  2. 选择一个Slave 升到 Master，其它的Slave挂到新提升的 Master

    • slaveof no one 提升为master

  3. 将其他 Slave 挂到新的 Master

    • slaveof 127.0.0.1 6382

==总结==

1、一个 master 可以有多个 slave

2、slave 下线，读请求的处理性能下降

3、master 下线，写请求无法执行

4、当 master 发生故障，需手动将其中一台 slave 使用 slaveof no one 命令提升为 master，其

它 slave 执行 slaveof 命令指向这个新的 master，从新的 master 处同步数据

5、主从复制模式的故障转移需要手动操作，要实现自动化处理，这就需要 Sentinel 哨兵，

实现故障自动转移

5.2 高可用 Sentinel 哨兵

Sentinel 系统有三个主要任务：

⚫ ==监控：Sentinel 不断的检查主服务和从服务器是否按照预期正常工作。==

⚫ 提醒：被监控的 Redis 出现问题时，Sentinel 会通知管理员或其他应用程序。

⚫ 自动故障转移：监控的主 Redis 不能正常工作，Sentinel 会开始进行故障迁移操作。将

一个从服务器升级新的主服务器。让其他从服务器挂到新的主服务器。同时向客户端提

供新的主服务器地址。

  • 开启哨兵模式 —- ./redis-sentinel ../sentinel26380.conf 

                                   ./redis-sentinel ../sentinel26382.conf

    ​ ./redis-sentinel ../sentinel26384.conf

6. 安全设置

6.1 密码设置

  • 修改redis.conf的配置文件

  • 找到 requirepass 行去掉注释，requirepass 空格后就是密码。（123456）

  • 访问有密码的 Redis 两种方式：

    ①：在连接到客户端后，使用命令 ==auth==密码，命令执行成功后，可以正常使用 Redis

    ②：在连接客户端时使用 -a 密码。例如 ./redis-cli -h ip -p port -a password
